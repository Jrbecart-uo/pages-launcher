Pages Launcher

Publish a static website to GitHub Pages - no command line, no Git.

1
Connect your GitHub account

Sign in once so Pages Launcher can create the repository and upload your files for you.

Create one at github.com → fine-grained tokens with Contents, Pages and Administration set to Read & write. It stays in this browser tab only.

Connected as
ready
2
Name the site and add your files

Use yourname.github.io for your main site, or any name for a project site (served under /that-name/).

Drop your site folder or files here
index.html should be at the top level
- or -

Keeps existing files and overwrites ones with the same path. Best for editing a page or two.

3
Publish

One click creates the repository (if needed), commits your files, and turns on GitHub Pages.

Your site is live at
⏱️Wait about 1 minute before opening the link - GitHub needs to publish your files first. A "404" right away is normal; it disappears once publishing finishes.
To update later: come back, drop the changed file(s), and Publish again.
How updates work: every Publish is one commit. Drop one file to change one page, or the whole folder to replace everything - history is kept automatically.
Runs entirely in your browser and talks directly to GitHub. Your files are never sent anywhere else.
