Connect GitHub

Paste a personal access token so the demo can create the repo and push your files. In the real version this is a single “Sign in with GitHub” button — users never see a token.

Make one at github.com → fine-grained tokens. Give it access to your repos with Contents: Read & write, Pages: Read & write, and Administration: Read & write (needed to create the repo). The token stays in this browser tab only.

First build can take ~1 minute. To update later: come back, drop the changed file(s), Publish again.
How updates work: every Publish is one Git commit. Drop one file to change one page, or the whole folder to replace everything. History is kept automatically.
